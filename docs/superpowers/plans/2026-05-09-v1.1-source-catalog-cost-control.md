# AnyKnews V1.1 Source Catalog And Cost Control Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the V1.1 source catalog, cost-aware refresh pipeline, compact disk cache, lazy source pagination, and drag-based source subscription management.

**Architecture:** Move source identity, cost, refresh policy, seed data, and catalog visibility into a single manifest layer, then derive the existing board model from it for backward compatibility. Replace full-board refresh with a planner that refreshes only subscribed source ids, honors low/medium/high cost rules, reads/writes compact disk cache, and exposes source-page APIs for lazy item loading. Keep browser preferences in `localStorage`, with the server receiving subscribed source ids on board fetches.

**Tech Stack:** Next.js App Router, TypeScript, React client state, native HTML drag events, Node `fs/promises` for disk cache, existing connector helpers in `lib/connectors/shared.ts`, `npx tsc --noEmit`, `npm run lint`, `npm run build`, and focused `npx tsx` verification scripts.

---

  - Store compact source cache and backoff records in memory plus `.cache/anyknews/source-cache.json`.

  - Decide page-open, manual, and pagination refresh eligibility from source id, cost, policy, cache status, and 10-minute freshness threshold.

  - Accept subscribed source ids and item limit, refresh only selected ids, use disk cache/backoff, return first 8 items for board loads, and expose source page loading.

  - Document V1.1 cache behavior, source subscription behavior, and relevant environment variables.
- Create `scripts/verify-source-catalog.ts`
  - Verify manifest uniqueness, default sources, V1.1 source coverage, and connector recipe status.
- Create `scripts/verify-refresh-policy.ts`
  - Verify low/medium/high source refresh decisions.

- [ ] **Step 1: Create reusable adapter factories**

Write `lib/connectors/adapters.ts`:

```ts
import type { SourceConnector } from "@/lib/connectors/types";
import { fetchJson, fetchText, extractLinksFromHtml, getPublicRssHubRoutes, parseRssItems, SOURCE_FETCH_LIMIT } from "@/lib/connectors/shared";
import { fetchDailyHotItems } from "@/lib/connectors/dailyhot";
import type { SourceConnectorRecipe, SourceManifest } from "@/lib/sources/types";

export function buildConnectorFromRecipe(source: SourceManifest): SourceConnector | undefined {
  const recipe = source.connector;

  if (source.catalogStatus === "catalog-only") {
    return undefined;
  }

  if (recipe.kind === "custom") {
    return undefined;
  }

  if (recipe.kind === "rss") {
    return {
      id: source.id,
      label: source.name,
      fetchItems: async () => parseRssItems(await fetchText(recipe.url), recipe.url)
    };
  }

  if (recipe.kind === "rsshub") {
    return {
      id: source.id,
      label: source.name,
      fetchItems: async () => {
        const routes = recipe.routes.flatMap((route) => getPublicRssHubRoutes(route));
        let lastError: unknown;
        for (const url of routes) {
          try {
            return parseRssItems(await fetchText(url), url);
          } catch (error) {
            lastError = error;
          }
        }
        throw lastError instanceof Error ? lastError : new Error(`${source.name} RSSHub routes failed`);
      }
    };
  }

  if (recipe.kind === "dailyhot") {
    return {
      id: source.id,
      label: source.name,
      fetchItems: () => fetchDailyHotItems(recipe.endpoint, source.name)
    };
  }

  if (recipe.kind === "html-list") {
    return {
      id: source.id,
      label: source.name,
      fetchItems: async () =>
        extractLinksFromHtml(await fetchText(recipe.url), {
          baseUrl: recipe.baseUrl,
          hrefIncludes: recipe.hrefIncludes,
          limit: SOURCE_FETCH_LIMIT,
          minTitleLength: recipe.minTitleLength
        })
    };
  }

  if (recipe.kind === "json-list") {
    return {
      id: source.id,
      label: source.name,
      fetchItems: async () => recipe.mapItems(await fetchJson<unknown>(recipe.url))
    };
  }
}
```

    return memoryCache;

  globalThis.__anyknewsDiskCacheLoaded = true;

  try {

    for (const entry of Object.values(payload.sources || {})) {
      memoryCache.set(entry.sourceId, entry);
    }

    return memoryCache;
  }

  return memoryCache;

  return [...memoryCache.values()];
}

export async function persistSourceCacheToDisk() {
  if (process.env.ANYKNEWS_DISABLE_DISK_CACHE === "true") {
    return;
  }

  const filePath = getCachePath();
  await mkdir(path.dirname(filePath), { recursive: true });
  const payload: CacheFile = {
    version: 1,
    sources: Object.fromEntries([...memoryCache.entries()].map(([sourceId, entry]) => [sourceId, sanitizeEntry(entry)]))
  };
  await writeFile(filePath, `${JSON.stringify(payload, null, 2)}\n`, "utf8");

import type { SourceManifest } from "@/lib/sources/types";

export type RefreshIntent = "page-open" | "manual" | "pagination" | "force";

export type RefreshState = {
  hasConnector: boolean;
  isCacheFresh: boolean;
  lastSuccessAt?: number;
  backoffUntil?: number;
  now: number;
};

export type RefreshDecision = {
  shouldFetch: boolean;
  reason:
    | "force"
    | "manual"
    | "pagination-miss"
    | "low-cost-stale"
    | "budget-threshold-expired"
    | "catalog-only"
    | "no-connector"
    | "cache-fresh"
    | "budget-threshold-fresh"
    | "backoff";
};

const budgetFreshnessMs = 10 * 60 * 1000;

export function decideSourceRefresh(source: SourceManifest, intent: RefreshIntent, state: RefreshState): RefreshDecision {
  if (source.catalogStatus === "catalog-only") {
    return { shouldFetch: false, reason: "catalog-only" };
  }

  if (!state.hasConnector) {
    return { shouldFetch: false, reason: "no-connector" };
  }

  if (intent === "force") {
    return { shouldFetch: true, reason: "force" };
  }

  if (intent === "manual") {
    return { shouldFetch: true, reason: "manual" };
  }

  if (state.backoffUntil && state.backoffUntil > state.now) {
    return { shouldFetch: false, reason: "backoff" };
  }

  if (intent === "pagination") {
    return state.isCacheFresh
      ? { shouldFetch: false, reason: "cache-fresh" }
      : { shouldFetch: true, reason: "pagination-miss" };
  }

  if (source.fetchCost === "low") {
    return state.isCacheFresh
      ? { shouldFetch: false, reason: "cache-fresh" }
      : { shouldFetch: true, reason: "low-cost-stale" };
  }

  const lastSuccessAt = state.lastSuccessAt ?? 0;
  if (state.now - lastSuccessAt > budgetFreshnessMs) {
    return { shouldFetch: true, reason: "budget-threshold-expired" };
  }

  return { shouldFetch: false, reason: "budget-threshold-fresh" };
}
```

      sourceOrder: [...nextSubscribed, ...current.sourceOrder.filter((id) => id !== sourceId && !nextSubscribed.includes(id))]
    };
  });
}

function unsubscribeSource(sourceId: string) {
  updatePreferences((current) => ({
    ...current,
    subscribedSourceIds: current.subscribedSourceIds.filter((id) => id !== sourceId),
    hiddenSourceIds: [...new Set([...current.hiddenSourceIds, sourceId])]
  }));
}

function handleSubscriptionDrop(event: React.DragEvent, insertIndex?: number) {
  event.preventDefault();
  const sourceId = event.dataTransfer.getData("text/source-id");
  if (sourceId) {
    subscribeSource(sourceId, insertIndex);
  }
}
```

Add to `README.md`:

```md
## V1.1 Source Catalog And Cost Control

AnyKnews keeps source preferences in browser localStorage. The server only fetches sources requested by the browser:

- Page open sends subscribed source ids and requests the first 8 items per source.
- Low-cost subscribed sources refresh when stale.
- Medium/high-cost subscribed sources refresh on page open only when the last successful refresh is older than 10 minutes.
- Card pagination calls the single-source item API and loads more items only for that source.
- Manual card refresh fetches only that source.
- The server writes a compact disk cache at `.cache/anyknews/source-cache.json`.
- The disk cache stores source metadata, title, summary, link, metrics, timestamps, diagnostics, and backoff state.
- The disk cache does not store full HTML, article bodies, images, or raw payloads.

Environment variables:

- `ANYKNEWS_DISK_CACHE_PATH`: override compact cache file path.
- `ANYKNEWS_DISABLE_DISK_CACHE=true`: disable disk cache.
- `ANYKNEWS_BOARD_ITEM_LIMIT`: default board items per source, default `8`.
- `ANYKNEWS_SOURCE_PAGE_ITEM_LIMIT`: default lazy page size, default `8`.
- `ANYKNEWS_FAILURE_BACKOFF_SECONDS`: failed source automatic retry backoff, default `300`.
- `DAILYHOT_API_BASE_URL`: optional DailyHotApi-compatible base URL.
```
